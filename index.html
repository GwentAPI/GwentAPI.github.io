Endpoints

GwentAPI currently support the following endpoints. More will follow.

URL Method Description
https://api.gwentapi.com/v0/ GET Return a mapping of resources name to href.
https://api.gwentapi.com/v0/cards/ GET Return all cards.
https://api.gwentapi.com/v0/cards/:id GET Return a specific card.
https://api.gwentapi.com/v0/cards/leaders GET Return all leader cards.
https://api.gwentapi.com/v0/cards/factions/:factionID GET Return all cards from the corresponding faction.
https://api.gwentapi.com/v0/cards/rarities/:idRarity GET Return all cards from the corresponding rarity.
https://api.gwentapi.com/v0/cards/:id/variations GET Return the variations of a specific card.
https://api.gwentapi.com/v0/factions/ GET Return all card factions.
https://api.gwentapi.com/v0/factions/:id GET Return a specific card faction.
https://api.gwentapi.com/v0/categories GET Return all categories.
https://api.gwentapi.com/v0/categories/:id GET Return a specific category.
https://api.gwentapi.com/v0/rarities/ GET Return all card rarities.
https://api.gwentapi.com/v0/rarities/:id GET Return a specific card rarity.
https://api.gwentapi.com/v0/groups/ GET Return all card groups.
https://api.gwentapi.com/v0/groups/:id GET Return a specific card group.
How is GwentAPI built?
  • The core of the service is built around a Go application
  • Proxied by Nginx
  • Hosted by DigitalOcean
  • With data stored inside a mongoDB cluster
  • Everything delivered to you with little delay thanks to CloudFlare
